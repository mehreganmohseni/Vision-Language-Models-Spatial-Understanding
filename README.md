# Vision-Language Models for Spatial Understanding

This project studies whether **Vision-Language Models (VLMs)** can act as useful judges for the compositional correctness of images generated by text-to-image diffusion models.

The project focuses on the fact that diffusion models can generate realistic-looking images, but may still fail at **compositional understanding**. Typical failures include:
- incorrect **2D spatial relations**
- incorrect **3D spatial relations**
- incorrect **object counts**
- failures in more **complex multi-object scenes**

To study this problem, the project compares:
- two image generators: **SDXL** and **FLUX**
- an automatic **proxy evaluation pipeline**
- multiple **VLM-as-a-Judge** models

The benchmark categories used in the project are:
- `spatial`
- `3d_spatial`
- `numeracy`
- `complex`

## Project Workflow

The project is organized as a sequence of four notebooks.

### `01_generate_images.ipynb`
This notebook is responsible for the **image generation stage**.

Main tasks:
- load benchmark prompts from the selected categories
- generate images using:
  - **SDXL**
  - **FLUX**
- use **3 seeds per prompt** in the main experiments
- save generated images and create indexing CSV files

Purpose:
- build the full dataset of generated images used in all later stages
- make the generation process reproducible and traceable

---

### `02_proxy_construction.ipynb`
This notebook builds the automatic **proxy evaluation pipeline**.

Main tasks:
- **Step 2**: use **OWL-ViT** for object detection and coarse relation/count reasoning
- **Step 3**: use **SAM3** for segmentation-based refinement of spatial relations
- build automatic proxy labels for:
  - object presence
  - spatial relations
  - numeracy / object counts

Purpose:
- provide an automatic reference baseline for evaluating large numbers of generated images

---

### `03_vlm_as_judge.ipynb`
This notebook evaluates the generated images using **multimodal VLM judges**.

Main tasks:
- run the following VLM judge models:
  - **Qwen2.5-VL**
  - **Qwen3-VL**
  - **Gemma 3**
- test multiple prompt variants for judge prompting
- ask the VLM to produce:
  - a **presence score** (1–5)
  - a **relation / count / composition score** (1–5)

Purpose:
- study whether VLMs can serve as semantic judges of generated images
- compare the judge outputs against the automatic proxy pipeline

---

### `04_final_demo_and_analysis.ipynb`
This notebook is the final integrative notebook of the project.

It has two roles:
1. **Compact end-to-end demo** on a small subset of prompts
2. **Final analysis** of the full experiment outputs

Main tasks:
- run a small demo pipeline:
  - subset selection
  - image generation
  - Step 2
  - Step 3
  - numeracy proxy
  - one VLM judge
- load the full saved outputs from the previous notebooks
- compute final analysis metrics such as:
  - mean proxy scores
  - mean VLM scores
  - normalized VLM scores
  - Pearson / Spearman correlation
  - Cohen’s kappa
  - weighted kappa
  - binary accuracy
  - JSON parse rate

Purpose:
- show a compact reproducible version of the full project pipeline
- provide the final interpretation of the full experimental results
